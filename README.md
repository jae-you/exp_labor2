The Invisible Engineer: AI Labor Conditions Simulator

이 프로젝트는 **개발자의 기술적 선택이 AI 도입 이후 노동자의 노동환경을 어떻게 바꾸는지**를 측정하기 위한 인터랙티브 실험 프로토타입입니다.

기존의 추상적 윤리 설문이 아니라, **클라이언트 요구사항**, **현장 노동자의 우려**, **개발자가 실제로 구현할 수 있는 제품 선택지**를 한 화면에 배치해 개발 업무의 현실성을 높였습니다.

## 실험 구조

실험은 2개 페이즈로 구성됩니다.

### Phase 1: 아키텍처 설계 시뮬레이션
- 총 7개 모듈을 순서대로 선택합니다.
- 각 모듈은 AICC 개발자가 실제로 받을 법한 기능 요청을 기반으로 구성됩니다.
- 선택 결과는 아래 3개 범주로 집계됩니다.
  - `노동자 자율성·권한`
  - `노동자 평가·관리`
  - `노동자 보호·안전`

### Phase 2: 설계 기술서 작성
- Phase 1에서 내린 선택을 바탕으로 세 가지 서술형 기획 문항에 답합니다.
- 데이터 통제권, 평가·관리 체계, 보호·안전 장치를 실무 기획서 수준으로 설명하도록 설계되어 있습니다.

## Phase 1 모듈 구성

1. 학습 데이터 수집 파이프라인
2. AI 초안 검토 워크플로우
3. 실시간 점수 랭킹 대시보드
4. AHT 목표치 반영 UI
5. AI·상담사 답변 로그 스키마
6. 악성 발화 대응 기능
7. 상담사 스트레스 신호 감지

## 설계 원칙

- 모듈 내용은 고객 경험보다 **노동자의 노동조건 변화**에 초점을 둡니다.
- 선택지는 모두 **개발자가 실제로 제품에 반영할 수 있는 구현 수준**으로 작성됩니다.
- 클라이언트의 요구가 주어진 뒤, 그 제약 안에서 개발자가 어떤 책임 있는 설계를 할 수 있는지 드러내는 것이 핵심입니다.

## 실행 방법

1. 설치
```bash
pip install -r requirements.txt
```

2. 실행
```bash
streamlit run app.py
```

## 주요 파일

- `app.py`: Streamlit 기반 실험 앱
- `sim.html`: 독립 실행형 시뮬레이터 UI
- `requirements.txt`: 실행 의존성
