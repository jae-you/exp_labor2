The Invisible Engineer: AI Labor Impact Simulator (V5.0)

이 프로젝트는 **"엔지니어의 기술적 의사결정이 노동 환경에 미치는 영향"**을 측정하기 위한 인터랙티브 웹 시뮬레이터입니다.

단순한 설문조사가 아닌, **[업무 메신저]**와 **[IDE 코딩 환경]**이 결합된 시뮬레이션을 통해 참가자(엔지니어)에게 실제 업무와 유사한 압박감과 피드백 루프를 제공합니다.

## 🧪 실험 구조 (Experiment Flow)

실험은 총 3단계의 **시나리오 기반 인터랙션**으로 진행됩니다.

### **Stage 1: Efficiency (CEO의 지시)**
- **등장인물:** CEO 최대표
- **상황:** 경쟁 우위를 위한 '속도'와 '비용 절감' 압박
- **과제:** 효율성 극대화를 위한 초기 아키텍처 설계 (V1)
- **예상 행동:** 상담원 연결 최소화, 빠른 처리 속도 중심의 코딩

### **Stage 2: Optimization (PM의 피드백)**
- **등장인물:** 기획팀장 박팀장
- **상황:** V1 배포 후 '정확도' 및 '재문의율' 문제 발생
- **과제:** 지표 개선을 위한 로직 고도화 (V2)
- **예상 행동:** 맥락 분석 강화, 라우팅 최적화

### **Stage 3: Sustainability (현장의 목소리)**
- **등장인물:** 상담원 김상담
- **상황:** 효율화된 시스템으로 인한 '노동 강도 증가'와 '감정 노동' 호소
- **과제:** 상담원 보호 및 지속 가능한 시스템을 위한 재설계 (V3)
- **예상 행동:** 욕설 필터링, 동적 휴식, 코칭 모드 등 인간 중심 기능 구현

## 🛠️ 기능적 특징 (Features)

1.  **Dual Interface:** 좌측 [업무 메신저]와 우측 [IDE 에디터]가 실시간으로 상호작용합니다.
2.  **Skeleton Prompting:** `{{...}}` 템플릿 방식을 사용하여, 참가자가 파라미터를 직접 고민하고 수정하게 유도합니다.
3.  **Active Validation:** 참가자가 내용을 수정하지 않으면 다음 단계로 넘어가지 않도록 검증합니다.
4.  **Immersive Narrative:** 실제 회사에서 겪을법한 대화 톤앤매너와 아바타를 사용하여 몰입감을 극대화했습니다.

## 🚀 실행 방법 (How to Run)

1.  **설치:**
    ```bash
    pip install -r requirements.txt
    ```

2.  **실행:**
    ```bash
    streamlit run app.py
    ```

## 📝 커스터마이징 (Configuration)

`app.py` 파일 하단의 `script` 태그 내에서 다음 변수를 수정하여 본인의 환경에 맞게 설정하세요.

* `GOOGLE_SCRIPT_URL`: 실험 데이터를 수집할 Google Apps Script 배포 URL
* `window.open('...')`: 실험 종료 후 연결될 Google Forms 설문조사 URL
